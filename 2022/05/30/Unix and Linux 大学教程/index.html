同时做不止一件事： I

  1. 多任务处理 (multitasking)

  6. 时间片 - 这一极短的时间被称为, 通常是 10 ms
  7. 时间片用完, 进程会挂起, Unix 调度器会决定下一个执行哪个进程
  8. 线程是一组运行在进程环境中的指令

新行字符的重要性

  1. 使用 ^J 标记每行的结束 - 新行字符
  2. 数据显示时以 MJ 结束
  3. 标准输入 - 键盘输入与读取文件的数据相同, 允许 Unix 以相同的方式读取数据而不必考虑数据源
  4. 返回字符 = ^M, 换行字符 = 新行字符 = ^J
  5. 一般而言, 每行文本必须以一个新行字符结束

比较文件

比较任意两个文件：cmp

  1. cmp file1 file2
  2. 若两个文件相同则不显示任何东西, 不相同则显示适当信息
  3. 可以比较任何类型的文件，音频视频等

比较有序文本文件：comm

  1. comm [-123] file1 file2

比较无序文本文件：diff

  1. diff [-bBiqswy] [-c|-Clines|-u|-Ulines] file1 file2
  2. diff 的目的是显示第一个文件修改为第二个文件所需遵循的指示
  3. diff 使用三个单字符指示
    • c (change) 改变
    • d (delete) 删除, 通常可以忽略 d 后面的数字
    • a (append) 追加
  4. 第一个文件中的行由 < 标识
  5. 第二个文件中的行由 > 标识
  6. 由 — 分隔两个文件

diff 使用的选项

  1. -i - 不区分大小写
  2. -w 和 -b 允许控制 diff 使用空白符的方式
    • w 忽略所有空白符
    • b 忽略空白符的数量
    • B 忽略所有空白行
    • q 两个文件不同时, 忽略所有细节

比较文件时的输出格式： diff、sdiff

  1. -c - 格式化输出 diff 信息
  2. -u - 格式化输出 diff 信息, 相比于 -c 而言, 没有重复行
  3. -y - 格式化输出 diff 信息, 文件并排
  4. sdiff - 等同于 diff -y
  5. sdiff [-bBilsW] [-w columns] file1 file2
    • -l - 如果拥有共同的行, 只显示左边
    • -s - 不显示两个文件中相同的行
    • -w - 改变展示的列宽

  3. 补丁 - 就是差分的一种, 用户使用 patch 程序应用该补丁
