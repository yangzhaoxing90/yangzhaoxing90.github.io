Servlet

本章介绍 Servlet API

1.1 Servlet API 概述

Servlet API 有四个Java包:

javax.servlet
javax.servlet.http
javax.servlet.annotation
javax.servlet.descriptor

javax.servlet 中的主要成员:

Servlet 接口定义了 Servlet 和 Servlet 容器的契约:

  1. Servlet 容器会把 Servlet 加载到内存中
  2. 并在 Servlet 实例中调用特定的方法

Servlet 相关实例:

  1. 请求 : ServletRequest
  2. 响应 : ServletResponse
  3. 环境 : ServletContext
  4. 配置信息: ServletConfig

1.2 Servlet

Servlet 接口定义了 5 个方法:

void init(ServletConfig config) throws ServletException
void service(ServletRequest req, ServletResponse res) throws 
ServletException, java.io.IOException
void destroy()
java.lang.String getServletInfo()
ServletConfig getServletConfig()

Servlet 生命周期:

init - 第一次请求 Servlet 时容器会调用这个方法, 后续的请求不再调用
service - 每次请求 Servlet 都会调用, 这里编写业务代码
destroy - 销毁 Servlet 时调用, 如容器关闭时

Servlet 的两个非生命周期方法:

getServletInfo - 返回 Servlet 的描述
getServletConfig() - 返回由 Servlet 容器传递给 init 方法 ServletConfig

1.3 编写基础的 Servlet 应用

应用程序的目录结构:

classes: Servlet 类和其他 Java 类都必须放在这
lib: 部署 Servlet 需要的 jar 文件, Servlet API jar 文件不需要部署在这, tomcat 已经包含了
JSP/HTML/图像等其他资源文件, 需要放在子目录下, 如 image 等等
若希望某个资源可以被 Servlet 访问, 而不能被用户访问, 则放入 WEB-INF 下

附录A见部署 servlet 的多种方法.

推荐使用打成 war 包的方式部署
war - 指的是以 war 作为扩展名的 jar 文件
使用 jar 来打成 war 包:

jar -cvf app_name.war *

1.4 ServletRequest

ServletRequest 中的部分方法:

public int getContentLength()
返回请求主体中的字节数, 如果不知道字节的长度, 则返回 1

public java.lang.String getContentType()
返回主体请求的 MIME 类型, 如果不知道类型, 则返回 null

public java.lang.String getParameter(java.lang.String name)
返回指定请求参数的值 

public java.lang.String getProtocol()
返回这个 HTTP 请求的协议名称和版本号

getParameter 是最常用的:

  1. 用来返回一个 HTML 表单域的值 (1.10)
  2. 可以用来获取查询字符串的值 (1.9)

1.5 ServletResponse

  1. getWriter 方法, 返回 java.io.PrintWriter, 默认采用 ISO-8859-1 编码
  2. getOutputStream, 用来传输二进制数据, 通常不用
  3. 在发送任何 HTML 标签响应之前, 应该先调用 setContentType 设置响应类型

1.6 ServletConfig

ServletConfig 可以通过 @WebServlet 或者 DD (部署描述文件 web.xml) 传递一个 Servlet 配置信息, 配置信息的初始参数是一个键值对

从 Servlet 内部获取初始参数的值:

java.lang.String getInitParameter(java.lang.String name)
返回初始参数值

java.util.Enumeration<java.lang.String> getInitParameterNames()
返回所有初始参数名称

ServletContext getServletContext()
(see 1.7)

有了 ServletContext 就可以共享能通过应用程序的所有资源访问信息, 促进 Web 对象动态注册

1.8 GenericServlet

实现了 Servlet 和 ServletConfig 这两个接口:

init 方法中赋值 ServletConfig
为 Servlet 所有方法提供默认实现
提供方法来包装 ServletConfig 中的方法

1.9 HTTP Servlet

HttpServlet :

继承 GenericServlet
HttpServletRequest 和 HttpServletResponse 表示 Servlet 请求和响应

HttpServlet 的两项特性 :

不覆盖 service 方法, 只覆盖 doGet 或者 doPost
用 HttpServletRequest 和 HttpServletResponse 代替 ServletRequest 和 ServletResponse

HttpServletRequest :

java.lang.String getContextPath()
返回表示请求 context 的 URI 部分 

Cookie[] getCookies()
返回一个 Cookie 对象数组 

java.lang.String getHeader(java.lang.String name) 
返回指定 HTTP 标头的值

java.lang.String getMethod()
返回发出这条请求的 HTTP 方法的名称 

java.lang.String getQueryString()
返回请求 URL 中的查询字符串 

HttpSession getSession()
返回与这个请求相关的 session 对象, 若没找到则创建新的 session 对象

HttpSession getSession(boolean create)
返回与这个请求有关的 session 对象, 若没找到, 且 create 为 true, 则创建新的 session 对象

HttpServletResponse :

void addCookie(Cookie cookie)
给这个响应对象添加 cookie

void addHeader(java.lang.String name, java.lang.String value)
给这个响应对象添加标头 

void sendRedirect(java.lang.String location)
发送响应代号, 将浏览器重定向到指定的位置

1.10 处理 HTML 表单

当用户提交表单时, 表单元素中输入的值会被当作请求参数发送至服务器:

html 输入域 (文本、密码等): 文本值会当作字符串发送服务器, 永远不为 null
html select 元素 : 发送选择的值作为字符串
多值选择 select multiple : 发送一个字符串数组, 需要用 getParameterValues 处理
复选框 : 被选中的复选框将字符 “on” 发送至服务器, 没被选中不发送任何内容
单选按钮 : 将选中的值发送, 没选中任何值, 则不发送
表单包含许多同名的输入元素: 所有值都会提交, getParameterValues 来获取
表单 action 缺失时, 会自动提交给发出请求的 URL

1.11 使用部署描述符

web.xml 并放入 WEB-INF 下

使用部署描述符的优势 :

能使用 @WebServlet 中不存在的属性, 如 load-on-startup, 使得 Servlet 在开始时就加载, 而不必等到使用时
如果需要修改配置值, 如 Servlet 路径, 就不需要重新编译 Servlet 类

Session 管理

本章讲解用于保存状态的 4 种方法:

网址重写 (URL rewriting)
隐藏域
cookie
HttpSession 对象

2.1 网址重写

是一种 Session 追踪技术, 需要将一个或多个 token 作为查询字符串添加到 URL 种

格式:

url?key-1=value-1&key-2=value-2...&key-n=value-n

2.2 隐藏域

只有当页面包含表单, 或者可以在页面中添加表单, 才适合使用隐藏域

input 标签的 hidden 属性, 使得服务器知道要修改的物品的 id

2.3 cookie

cookie 是自动在 Web 服务器和浏览器之间来回传递的一小块信息

对于 Web 浏览器而言, 每台 Web 服务器最多可以支持 20 个 cookie

cookie 的不足之处在于, 用户可以修改浏览器设置来拒绝接收 cookie

javax.servlet.http.Cookie
httpServletResponse.addCookie(cookie) 将一个 cookie 发送给浏览器
httpServletRequest.getCookies()
没有办法能直接删除 cookie, 为了删除 cookie 只能创建一个同名的 cookie, 将 maxAge 设置为 0

2.4 HttpSession 对象

所有 Session 追踪技术中 HttpSession 是最强大的

用户可以没有或者有一个 HttpSession, 并且只能访问他/她自己的 HttpSession

HttpSession 是用户第一次访问某个网站时自动创建的

通过在 HttpSession 中调用 getSession 方法

HttpSession getSession() 
返回当前的 HttpSession, 如果没有, 则创建一个新的返回

HttpSession getSession(boolean create)
getSession(false) - 返回当前的 HttpSession, 如若没有返回 null
getSession(true) = getSession()

void setAttribute(java.lang.String name, java.lang.Object value) 
将一个值放入 HttpSession 中 (放入 HttpSession 中的值是放入内存中的)
放入 HttpSession 中的值不一定是 String, 也可以是任意 Java 对象, 只要实现了 java.io.Serializable 接口即可

java.lang.Object getAttribute(java.lang.String name) 
获取 HttpSession 中保存的对象的值 

java.util.Enumeration<java.lang.String> getAttributeNames() 
迭代 HttpSession 中的所有属性名

java.lang.String getId()
获取 HttpSession 标识符

int getMaxInactiveInterval()
返回用户离开的秒数 

void setmaxInactiveInterval(int seconds)
为个别 HttpSession 的 Session期限 设置一个不同的值

HttpSession 的值不能发送至客户端, 而是 Servlet 容器为它所创建的每一个 HttpSession 生成唯一标识符
并将这个唯一标识符作为一个 token 发送给 浏览器, 一般是名为 JSESSIONID 的 cookie
或者作为一个 jsessionid 参数加载 URL 后面
invalidate 方法, 强制 session 过期; 默认情况下, HTTPSession 在用户静默一段时间后过期 (16 章)

JSP

JSP 的注释、隐式对象, 三个句法元素 (指令、脚本元素、动作)

JSP 可以用标准语法或者 XML 语法进行编写 (只介绍标准语法)

3.1 JSP 概述

JSP 其实是一个 Servlet

JSP 不需要编译, 可以用任何文本编辑器来编写

JSP 页面是在 JSP 容器里运行的, Servlet 容器一般也是 JSP 容器

第一次请求 JSP 页面时, Servlet/JSP 容器要做两件事 :

将 JSP 页面转换为 JSP 页面实现类 (javax.servlet.jsp.JspPage 或 javax.servlet.jsp.HttpJspPage)
若转换成功, Servlet/Jsp 容器将会编译 Servlet 类

对于后续的请求, Servlet/JSP 容器会查看这个 JSP 页面自最后一次转换以来是否修改过, 若修改过,
就会重新转换、重新编译并执行

第一次调用 JSP 总是比后来的请求的响应时间更长, 可以这样解决 :

配置应用程序, 在程序启动时就调用所有 JSP 页面
预先编译 JSP, 让他们以 Servlet 的方式进行部署

JSP 的 API:

javax.servlet.jsp 核心类和接口
javax.servlet.jsp.target 用于开发定制标签的类型
javax.el 为 Unified Expression Language 提供 API
javax.servlet.jsp.el 提供 Servlet/JSP 容器必须支持的类, 以便支持 JSP 中的 Expression Language

元素对于 JSP 转换器有特别的含义:

<% - 代表 java 代码的开始
%> - 代表 java 代码的结束
不属于元素的其他内容都是模板数据, 如 html 标签和文本
tomcat 的 work 下看到 jsp 转换的文件, 是一个 Servlet, 并且继承了 org.apache.jasper.runtime.HttpJspBase
<%…%> 块称为 scriplet (3.5 节)
page 指令可以导入 java 库 (3.4 节)

3.2 备注

两种备注:

  1. Jsp 备注, 注明该页面的作用
  2. HTML/XHTML 备注, 将被发送至浏览器
<%-- retrieve products to display --%>
这是一个 JSP 备注, 它不能嵌套使用

<!-- [comments here] -->
HTML/XHTML 备注

3.3 隐式对象

javax.servlet.jsp.PageContext 可以通过方法访问 request、response 等
但是意义不大, PageContext 还允许利用 Expression Language 访问哪些对象 (第 4 章)

public abstract void setAttribute(java.lang.String name, java.lang.Object value, int scope) 
PageContext 允许在某个范围内存取数据

scope 可以有四个值：

  1. PAGE_SCOPE - 值只能在同一个 jsp 页面中使用
  2. REQUEST_SCOPE - HttpRequest 范围
  3. SESSION_SCOPE - HttpSession 范围
  4. APPLICATION_SCOPE - ServletContext 范围

jsp 隐式条件:

  • 隐式对象 out 引用 javax.servlet.jsp.JspWriter
  • 类似于 HttpServletResponse 中调用 getWriter() 得到的 Java.io.PrintWriter
  • JSP 页面默认返回 text/html 类型

3.4 指令

指令是一种 JSP 句法元素, 指示 JSP 转换器应该如何将一个页面转换为 Servlet 命令

page 指令:

告诉 JSP 转换器隐式对象 out 应该使用多大的容量空间
要使用哪些内容类型
要导入哪些 java 类型

语法:

<%@ page attribute1="value1" attribute2="value2" ... %>
@ 和 page 之间的空格是可选的 
attribute1 和 attribute2 是 page 指令的属性

page 指令的属性列表:

  1. import - 指定要导入的 java 类型 (隐式导入的包: java.lang、javax.servlet、javax.servlet.http、javax.servlet.jsp)
  2. session - 值为 true, 表示参与页面 session 管理, 否则为 false. 默认值为 true, 若之前没有 HttpSession, 则会自动创建
  3. buffer - 指定 out 的缓存区大小, 以千字节为单位 (kb), 值为 none 表示不使用缓存, 直接输出 PrintWriter
  4. autoFlush - 默认值为 true, 缓存满时自动刷新输出. 值为 false, 刷新缓存需要手动调用 response.flush(), 否则缓存满了抛出异常
  5. isThreadSafe - JSP 作者建议不要使用这个属性. 表示页面的线程安全级别.
  6. info - 指定所生成 Servlet 的 getServletInfo 的返回值
  7. errorPage - 表示负责处理该页面可能出现错误的页面
  8. isErrorPage - 表示该页面是否负责处理错误
  9. contentType - 指定 response 的内容类型, 默认值为 text/html
  10. pageEncoding - 指定页面的字符编码, 默认值为 ISO-8859-1
  11. isELIgnored - 表明是否忽略 EL 表达式.
  12. language - 指定页面所用的脚本语言, 默认是 java, JSP2.2中唯一有效值
  13. extends - 指定这个 JSP 页面的实现类必须扩展超类, 很少使用该属性
  14. deferredSyntaxAllowedAsLiteral - 指明是否允许用字符序列 (作为该页面和编译单元的 String 字面值), 默认值是 false
  15. trimDirectiveWhitespaces - 是否输出时删除只包含空格的模板文本, 默认值为 false, 即不删除

page 指令可以放在页面的任何位置, 但是包含 contentType 或者 pageEncoding 就要放在所有模板数据之前

include 指令:

可以将另一个页面的内容放入当前 JSP 页面
一个 JSP 可以有多个 include

include 语法:

<%@ include file="url" %>

@ 和 include 之间的空格可选, url 表示文件的相对路径
若 url 以 / 开头, 则在服务器中被解读为绝对路径 
否则被解析为相对当前 jsp 文件所在位置的路径

按照规范, include 文件的拓展名为 jspf, 表示为 jsp fragment.

3.5 脚本元素

第二种 JSP 句法元素是脚本元素, 它将 Java 代码合并成一个 JSP 页面,
脚本元素有 3 种类型 :

Scriptlet
声明
表达式

Scriptlet 是一个 Java 代码块, 以 <% 开始 %> 结束

表达式 :

Today is <%=java.util.Calendar.getInstance().getTime()%>
结果会传给 out.print()

声明 :

<%!
    public String getTodayDate() {
        return new java.util.Date();
    }
%>
声明了一个 getTodayDate 方法

可以利用声明覆盖 Servlet 的 init 和 destroy 方法:

jspInit - JSP 页面初始化时调用
jspDestroy - JSP 页面要被销毁时调用

关闭脚本元素
随着 JSP 2.0 的发展, 推荐使用 EL 表达式来访问服务端对象, 而不是在 JSP 写 Java 代码

<jsp-property-group>
    <url-pattern>*.jsp</url-pattern>
    <scripting-invalid>true</scripting-invalid>
</jsp-property-group>

3.6 动作

第三种句法元素是动作 (Action), 它们被编译成执行某个操作的 Java 代码
本章讨论能被 JSP 容器支持的标准动作

useBean :

创建一个与某 Java 对象相关的脚本变量

<jsp:useBean id="today" class="java.util.Date" />
将 today 变量与 Date 对象关联起来

setProperty 和 getProperty :

setProperty 在 Java 对象中保存一个属性
getProperty 获取一个 Java 对象的属性

<jsp:useBean id="employee" class="app03a.Employee"
<jsp:setProperty name="employee" property="firstName" value="Abigail"/>
First Name: <jsp:getProperty name="employee" property="firstName" />

include :

动态的包含另一个资源, 可以包含另一个 JSP 页面、一个 Servlet 或者 一个静态的 HTML 页面

<jsp:include page="jspf/menu/jsp">
    <jsp:param name="text" value="How are you?">
</jsp:include>

forward:

将当前页面跳转到另一个不同资源

<jsp:forward page="jspf/login.jsp">
    <jsp:param name="text" value="Please login">
</jsp:forward>

错误处理:

利用 page 的 isErrorPage 属性
其他页面使用 errorPage 指定, 未捕捉的异常会自动跳转 error 页面

EL

JSP 2.0 中最重要的特性之一 Expression Language (EL) , JSP 设计者可以用它访问应用程序

EL 最早出现在 JSTL 1.0 中, JSP 2.0 以来, 不需要 JSTL 就可以使用 EL

本章只关注针对非 JSF 开发者的 EL

4.1 EL 语法

${expression}

两个表达式相连

${4+4}${5+5} -> 810

EL 在某个标签属性中, 将会运算这个表达式

<my:tag someAttribute="${2+2}"/> -> <my:tag someAttribute="4">
若想保留 ${, 则通过 \${ 来转义

保留字 (不能用作标识符) :

[] 和 . 运算:

用于访问某个对象的属性

${object["propertyName"]}
${object.propertyName}

如果 propertyName 不是一个合法的 Java 变量名称, 则只能使用 []

${header["accept-language"]}

级联访问属性:

${pageContext["request"]["servletPath"]}

访问 HttpSession, 可以使用语法 :

${pageContext.session}
${pageContext.session.id} - 输出 token

运算规则:

EL 表达式的运算是从左到右的
具体规则 p93

4.2 访问 JavaBean

${beanName["propertyName"]}
${beanName.propertyName}

4.3 EL 隐式对象

EL 隐式对象帮助访问 Jsp 隐式对象

pageContext:

表示 javax.servlet.jsp.PageContext
它包含所有 JSP 的隐式对象

${pageContext.request}
${pageContext["request"]}
可以获得 ServletRequest 

${pageContext["request"]["method"]}
可以获取请求方法

initParam:

${initParam.password}
${initParam["password"]}
可以获得 context(PageContext) 的参数 password

param:

${param.userName}
${param["userName"]}
用于获取请求参数

paramValues:

包含所有请求参数的参数名作为键的 Map 
每个键的值都是字符串数组, 包含指定参数名称的所有值 

${paramValues.selectedOptions[0]}
${paramValues.selectedOptions[1]}
获得 selectedOptions 参数的第一个和第二个值

header:

表示包含所有请求标头的 Map 

${header["accept-language"]}
${header.connection}

headerValues:

表示一个包含所有请求头并以标头名称作为键的 Map 
但是它的值是字符串数字 

${headerValues["accept-language"][0]}

cookie:

cookie 对象表示包含当前 HttpServletRequest 中所有 cookie 的 Map 

${cookie.jsessionid.value}
获取 jsessionid 的值 
${cookie.jsessionid.path}
获取 jsessionid 的路径值

applicationScope、sessionScope、requestScope 及 pageScope:

可以获取相对应范围的变量值 

${applicationScope.myVar}

有作用范围的对象指的是放在以下这些对象中作为属性的对象:

PageContext
ServletRequest
HttpSession
ServletContext

有作用范围的对象可以直接用 EL 表达式访问:

${today}

访问顺序为:
1. PageContext  
2. ServletRequest  
3. HttpSession  
4. ServletContext

4.4 使用其他 EL 运算符

算术运算符:

加法/减法/乘法/除法(/ 和 div)/模(% 和 mod)

关系运算符:

等于/不等于/大于/大于或等于/小于/小于或等于

逻辑运算符:

AND/OR/NOT

条件运算符:

${statement? A:B}

empty 运算符:

用来校验一些值是否为 null 或 空 

${empty x}
若 x 为 null -> true  
若 x 是长度为 0 的字符串 -> true 
若 x 是空 Map、空数组、空集合 -> true
否则 -> false

JSTL

JSTL 是一个定制标签库, 用于解决一些常见的问题

迭代一个集合、条件测试、XML处理、数据库访问和数据操作等

本章主要讨论 JSTL 中最重要的标签, 尤其是访问限制域对象, 迭代集合及格式化数字和日期的那些标签

5.1 下载 JSTL

5.2 JSTL 类库

JSTL 类库可以分为 5 类 (JSTL 1.2)

在 JSP 页面中使用 JSTL 必须使用 taglib 指令:

<%@ taglib uri="uri" prefix="prefix" %>

<%@ taglib uri="http://java.sun.com/jsp/jstl/core" prefix="c" %>
为了使用 Core 类库, 必须在 JSP 开头处这样声明

注意事项:

标签的属性名称后面加 (*) 表示属性是必须的
(+) 表示该属性的 rtexprvalue 元素为 true, 表示是动态的, false 表示只能用静态字符串

5.3 通用动作指令

Core 类库中的用来操作限域变量的 3 个通用动作指令: out、set 和 remove.

out 标签:

out 标签对表达式进行运算, 并将结果输出到 JspWriter

out 标签的两种语法形式

<c:out value="value" [escapeXml="{>true|false}"] [default="defaultValue"] />
<c:out value="value" [escapeXml="{>true|false}"]>
    default value 
</c:out>

[] 表示可选属性, 加了>的值是默认值

set 标签:

创建一个字符串和引用该字符串的一个限域变量
创建一个限域变量, 并引用已经存在的某个限域变量
设置限域变量属性
使用 set 标签设置的限域变量整个 JSP 页面都可以使用

set 标签语法有 4 种形式 

<c:set value="value" var="varName" [scope="{>page|request|session|application}"] />
第一种形式, value 可以是字符串, 或者是现有的限域对象

<c:set var="varName" [scope="{>page|request|session|application}"]>
    body content
</c:set>
第二种形式, 允许主体中有 jsp 代码

<c:set target="target" property="propertyName" value="value" />
第三种形式, 设置限域对象的属性值, target 中用 EL 表达式 

<c:set target="target" property="propertyName">
    body content 
</c:set>
第四种形式, 值作为主体内容传递

remove 标签:

利用 remove 标签删除限域对象

<c:remove var="varName" [scope="{>page|request|session|application}"] />
注意: 限域变量的引用并没有删除, 其他引用该对象的变量还可以正常工作

5.4 条件式动作指令

JSTL 中有四个条件式动作指令:

  1. if
  2. choose
  3. when
  4. otherwise

if 标签:

<c:if test="testCondition" var="varName" [scope="{>page|request|session|application}"] />
第一种形式, 计算 test 的值, 利用 var 属性设置限域变量名引用 Boolean 对象 

<c:if test="testCondition" [var="varName"] [scope="{>page|request|session|application}"]
    body content 
</c:if>
内容是 jsp, 如果测试条件为 true, 它就会得到处理

choose、when 和 otherwise 标签:

<c:choose>
    <c:when test="condition1">
        content1 
    </c:when> 
    <c:when test="condition2">
        content2
    </c:when>
    <c:otherwise>
        content3
    </c:otherwise>
</c:choose>

5.5 iterator 动作指令

迭代一个对象集合, iterator 非常好用

JSTL 提供了两个标签:

forEach
forTokens

forEach 标签:

<c:forEach [var="varName"] begin="begin" end="end" step="step">
    body content 
</c:forEach>
将 body 的内容重复多次 

<c:forEach items="collection" [var="varName"] [varStatus="varStatusName"] [begin="begin"] [end="end"] [step="step"]>
    body content 
</c:forEach>
主体内容为 JSP 

还可以利用 forEach 来遍历一个 Map

<c:forEach var="mapItem" items="map">
    ${mapItem.key} : ${mapItem.value}
</c:forEach>

forTokens 标签:

<c:forTokens items="stringOfTokens" delims="delimiters" [var="varName"] [varStatus="varStatusName"] [begin="begin"] [end="end"] [step="step"]>
    body content 
</c:forTokens>
主体内容为 JSP

5.7 函数

JSTL 还定义了一组函数, 可以用在 EL 表达式中

<%@ taglib uri="http://java.sun.com/jsp/jstl/functions" prefix="fn" %>

调用的格式:

${fn:functionName}

大多数都是用于字符串的函数:

contains(string, substring) -> Boolean
containsIgnoreCase(string, substring) -> Boolean
endsWith(string, suffix) -> Boolean
escapeXml(string) -> String
indexOf(string, substring) -> Integer
join(array, seperator) -> String
length(input) -> Integer
replace(string, beforeSubstring, afterSubstring) -> String
split(string, delimiters) -> String[]
startsWith(string, prefix) -> Boolean
substring(string, beginIndex, endIndex) -> String
substringAfter(string, substring) -> String
substringBefore(string, substring) -> String
toLowerCase(string) -> String
toUpperCase(string) -> String
trim(string) -> String
