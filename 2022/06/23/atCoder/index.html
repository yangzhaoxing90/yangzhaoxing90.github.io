  1. 看仔细题目含义
  2. 循环内部的循环判断, 通过建立函数的方式, 代码可读性更好

43 BC


  1. string 使用 push_back 和 pop_back 往末尾添加或删除字符
  2. 需要注意, pop_back 在 string 为空的时候会报错
  3. INT_MAX 属于 #include<climits> 头文件
  4. 使用 cin >> v[i], 比 v.push_back(t) 要干净一些

44 BC


  1. 循环对于偶数奇数的判断, 可以通过结果为 0 或者 1 来判断 (求和为0或求和为1)
  2. C 问题是 dp 的三维背包问题, dp[i][j][k] 表示前 i 个数取 j 个数和为 k 的方案数, 难点在于考虑到个问题
  3. 递推方程为: dp[i][j][k] = dp[i-1][j][k] + … + dp[i-1][0][k-w[i]]
  4. 初始条件为: dp[i][0][0] = 1, dp[1][1][x] = dp[0][1][x] + dp[0][0][0] = 1, 因此 dp[i][0][0] 必须设置为 1
  5. 考虑三维坐标轴使用滑动数组, 遍历的方向为从前往后, 然后从面的右上往左下

45 BC


gacchigachi58 - 参考
  1. i & j 作为判断条件, 返回 i 和 j 的与的结果作为判断条件
  2. 选择一个数字和上个数字合并或者单独, 有两种状态, 共能选择 n - 1 次，因此总共有 1 << (n - 1) 种情况
  3. 难点在于 i & (1 << j) 的判断理解上, i 从 1 一直增大, 最后一位数据如同开关一般在 0 与 1 之间切换
  4. 1 << j 就是 1 在 j 的位置上
  5. 建议直接记忆 i & (1 << j) 可以表示 两个状态 取所有 情况

46 BC


  1. 排列组合的思想
  2. a1 <= m * a2, 并且 t1 <= m * t2, 因此 m 的取值为 max(ceil(a1/a2), ceil(t1/t2))
  3. 但是用 ceil 会出问题, 会导致精度问题 (原因不明), 因此最好使用代数的方法向上取整
  4. a1/a2 + 1, 对于 a1 > 0 且 a2 > 0 大多数情况起作用, 当 a1 = a2 时，会出现问题
  5. 因此使用 (a1 - 1) / a2 + 1 来表示向上取整, 当 a1 = a2 时, 结果正确为 1
  6. 当 a1 < a2 时, 结果正确为 1, 特别的 a1 = 1, 0/a2 + 1 = 1
  7. 当 a1 > a2 时, 结果正确为 [a1/a2]+1, 特别的, a1 = a2+1, a2/a2+1 = 2 结果正确
  8. 因此使用 (a1 - 1) / a2 + 1 来表示向上取整对于 a1 > 0 且 a2 > 0 是正确的

47 BC


  1. 有了巧妙的思路后再进行编码
  2. C 题：考虑棋子有几个需要翻转的状态, 就结束了 …

48 BC


1. 论当 a > 0 时, (a - 1) / x + 1 的妙用, 可以消除 a / x = 0 带来的问题 2. 贪心算法, 考虑局部最优, 两个相邻的数相加大于给定的数, 一定要让第二个数减小, 这样可以让后面的数尽可能的大, 这样所需要减少的数越小 3. 本题的贪心逻辑是这样的, 如果是偶数个数, 那么不用贪心, 减少的数是确定的, 因为任意相邻的两个数的和是确定的 4. 问题在于奇数个数, 需要从后往前推, 需要第 N-1 个数尽可能的小, N-2 个数就需要尽可能的大, N-3 个数就需要尽可能的小, ... , 符合条件的情况下, 第一个数就需要尽可能的大, 第二个数就需要尽可能的小 5. 所以只需要确保第一个数是 min(x, a[0]), 然后顺推就可以了

49 BC


1. (i+1)/2 相当于将 1 和 2 都设置为 1, 3 和 4 转换为 2, 依次类推 2. 各个单词重复, 所以导致了难度, 把单词倒序, 问题就简单了 3. 要判断字符串的子字符串是否相等, 可以用空字符串加上一个字符串, 去掉 substr 的函数方法

50 BC


1. 先做出求和, 再对单点替换数值进行计算, 效率更高 2. 奇数-奇数一定 = 偶数, 偶数 - 偶数 一定 = 偶数, 偶数 - 奇数 或 奇数 - 偶数 一定 = 奇数
